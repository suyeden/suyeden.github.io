blog-mode.el

Emacs 上で動く、個人用途用の簡易ローカルWiki風プログラムです。
メモや日記等の用途に使えます。
使用するには Emacs のインストールが必要です。

ソースコードの閲覧およびプログラムのダウンロードはこちらから

以下、導入方法や使用方法（GitHub の README の日本語訳）

導入方法

まず Emacs をインストールしてください。
Emacs をインストールしたら、Org-mode が使えることを確認してください。
（おそらく何もしなくてもデフォルトで使える状態になっていると思います。）
なお、Emacs や Org-mode の導入方法や基本的な使用方法については、ネット上にたくさん情報が転がっているので、自分で探してみてください。

Linux ユーザーの方は、apt 等のパッケージ管理システムで導入した Emacs ではバージョンが古く想定外の挙動を示す場合があるため、自分で最新版（少なくとも GNU Emacs 27.1 以上）をビルドするようにしてください。

Emacs と Org-mode が使える状態であれば、blog-mode.el ファイルを ～/.emacs.d/lisp/ のような適当なディレクトリ下に配置して、.emacs や init.el のような設定ファイルに以下のようなコードを追加してください。

(add-to-list 'load-path "~/.emacs.d/lisp")
(load-library "blog-mode.el")

使用方法

blog-mode を開始するには、C-c b（Ctrl + c を押してから b を押す）を押してください。
初回起動の場合はホームディレクトリ下に /org/blog ディレクトリが作成されます。
以降、blog-mode を用いて記録されたファイル（実体は org ファイル）はこのディレクトリ下に保存されます。
すでに開いているファイルに対して blog-mode を適用するには、「M-x blog-mode（Alt + x を押してから blog-mode と入力）」を使うこともできます。

ヘルプを開くには C-c C-h を押してください。
q でヘルプから抜けることができます。
使い方が分からなくなったら、とりあえずヘルプ開けばどうにかなるかもです。

その他 blog-mode で使えるキーバインドは以下の表の通りです。

キーバインド 機能
C-c b blog-mode を開始する
C-c C-h ヘルプを開く
C-c M-r blog-mode を再起動する（blog-mode がおかしくなった時に使用）
M-RET 見出しを挿入する
C-c n 新たなブログトピック（実体は新たな org ファイル）を作成し、ファイルへのリンクを貼る
C-c C-o リンク先へジャンプする
C-c C-LEFT １つ前のトピック（リンクを開く前のトピック）へ戻る
C-c r ブログトピックの名前を変更する
C-M-d ブログトピックを削除する
C-c C-e h H/h/o カレントバッファに開いているトピック（org ファイル）を HTML にエクスポートする
C-c x 新たに作成された、あるいは修正された全てのトピック（org ファイル）を HTML にエクスポートする
C-c e blog-mode を終了する
M-LEFT or M-RIGHT 見出しのレベル（深さ）を変更する
M-UP or M-DOWN 見出しを前後に移動する（並び替える）
C-c C-l 保存されたリンクを挿入する
S-TAB or C-u C-i 全ての見出しを隠す/展開する
TAB or C-i 現在行の見出しを隠す/展開する
C-c C-, コードブロックを挿入する
C-c ' コードブロックを編集する
C-M-i ソースコードブロック上で、サポートされている言語のリストを表示する
C-c M-s 選択範囲内の全ての行の先頭に、指定量の半角空白を挿入する
C-j 行頭の空白を考慮して改行する
C-c C-n/p 前/後の見出しへ移動する


これらの他にも org-mode でサポートされているキーバインドや文法は大体使えるはずです。

エクスポートした HTML にスタイルシートを適用したい場合は、～/org/blog/css/style.css ファイルを編集してください。

また初回起動時にブログのタイトルや著者名、使用言語を聞かれますが、一度設定した後にこれらを変更したい場合は、index.org を直接編集してください。

全てをリセットしてやり直したい場合は、ホームディレクトリ下にある /org/blog ディレクトリ内のファイルを全て削除、あるいは /org/blog ディレクトリそのものを削除してください。
