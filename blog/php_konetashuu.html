セッション

sessionstart で開始・再開。
端末情報は セッションID（sessionid()） や セッション名（sessionname()） によって判別される。
$SESSION('Id') や $SESSION('Name') は単にセッションで登録された（自分自身で登録した）「セッション変数」であり、サーバーによって与えられる一意の値・識別子ではない。

要点

  • セッションは、サイトを訪れた 個々のユーザーのデータ を 個別に管理する機能 を提供する
  • セッションデータは、 セッションデータコンテナ に格納される
  • セッションデータは、 一意な識別子(ユニークなID)セッションIDによって管理 される
  • サーバー側の セッションID とブラウザ側のクッキーに格納された PHPSESSID が一意な同一のIDであり、 ユーザーを関連付け識別する
  • セッションを利用するには、 sessionstart() 関数と セッション変数 の $SESSION (スーパーグローバル変数の1つ)を使用する
  • $SESSION は PHPの定義済み変数(=スーパーグローバル変数) の1つ
  • $SESSION は 現在のセッションに登録されている値の変数
  • $SESSION は 連想配列 として使用する
  • $SESSION は、関数やメソッドの内部で使用する場合、 global $SESSION; とする必要がない
  • sessionid() 関数 セッションIDを確認できる
  • sessionname() 関数 セッション名を確認できる
  • sessionregenerateid() 関数 セッションIDを再発行できる
  • sessionunset() 関数 セッション変数を開放できる
  • sessiondestroy() 関数 全セッションデータを破棄できる
  • セッションは ラウザを閉じた時点で終了 する
  • php.iniのsession.cookielifetimeで有効期限を設定 できる

メールの送信

mbsendmail 関数を使う

// メールの使用言語を設定する
mb_language("Japanese");
// PHPの内部で使用する文字コードを設定する
mb_internal_encoding("UTF-8");

$to = 'メールアドレス';
$subject = '件名';
$body = '本文';

// メールの送信、送信できたら true を返す
$result = mb_send_mail($to, $subject, $body);

if ($result) {
    echo 'メールを送信しました';
} else {
    echo 'メール送信に失敗しました'
}

エラーを表示させる方法

ログファイルが Linux では /opt/lampp/logs/errorlog にある。
ローカルホストで開発中にエラーを表示したい場合はプログラム先頭に次のように書く。

<?php
ini_set('display_errors', 'on');
// E_ALL はすべてのエラー、例えば Notice だけの場合は E_NOTICE
error_reporting(E_ALL);

本番運用する場合はオフにしておくこと。

Apache HTTP Server, MySQL の起動

  1. 管理者権限でXAMPPコントロールパネルのアイコン（右下のタスクバー等にある）をクリック
    管理者権限で開かなければ、終了の際に "Error:Cannot create file "C:\xampp\xampp-control.ini". Access denied. のようなエラーメッセージが出る。
  2. 「Apache」と「MySQL」の横にある[Start]ボタンをそれぞれクリック
    正常に起動すると、ApacheとMySQLの背景が緑色に変化する
  3. 終了する際はコントロールパネル横の「Quit」を押す。

公開フォルダ（WWWサーバーのルートディレクトリ）

XAMPPコントロールパネルの[Explorer]ボタンをクリックすると、XAMPPをインストールしたフォルダが表示される。
その中に [htdocs] というフォルダがある
このフォルダは公開フォルダと呼ばれ、プログラムやWebページを置く場所となる。（フォルダの右クリック→送る→デスクトップで、ショートカットの作成が可能）
公開フォルダ直下においたファイルは http://localhost/ファイル名 でアクセスできる。
公開フォルダ以下にフォルダを作成し、そこにファイルを置きたい場合は http://localhost/フォルダ名/ファイル名 のようにスラッシュで区切られる。

PHPの日本語環境の設定

「php.ini」を編集してphpの日本語環境を設定する。また文字コードも「UTF-8」に統一する。
php.iniの保存場所は「C:\xampp\php」。
「php.ini」をテキストエディタで開き、次表の通りに変更する。なお、行の先頭に「;」がある場合は「;」を削除する。

ディレクティブ 設定値
date.timezone Asia/Tokyo
defaultcharset "UTF-8"
mbstring.language Japanese
mbstring.internalencoding UTF-8
mbstring.substitutecharacter none;

MySQLへの接続

MySQL5.7以降では、rootユーザーでログインする⼿順が少し変わっている。これは、セキュリティに対してより厳しくなったことによる変更である。
rootユーザーには必ずパスワードを設定しなければいけない仕様になっており、MySQL8.0以降にrootユーザーでログインするためにはパスワードの初期設定が必要。

phpMyAdminでMySQLの「root」パスワードを設定し、そのパスワードを「config.inc.php」に記述する。

  1. ブラウザで http://localhost/phpmyadmin/ にアクセスし、phpMyAdminを開く。
    なお、言語が日本語になっていない場合は、日本語に変更する。
  2. 「特権」タブをクリックし、ホストが「localhost」の「root」の「特権を編集」をクリックする。
  3. 「パスワードを変更する」の「パスワード」と「もう一度入力してください」にパスワードを入力し、[実行する]ボタンをクリックする。
  4. 「config.inc.php」をテキストエディタで開く。
    「config.inc.php」の保存場所は C:\xampp\phpMyAdmin 。
  5. 「$cfg['Servers'][$i]['password'] = ";」という記述を検索し、「"」内に 4 で設定したパスワードを記述し、上書き保存する。

上記設定を行った上で、MySQLへの接続を行う。
接続を行うには、管理者権限でXAMPPコントロールパネルを開き、コントロールパネル横の「Shell」をクリックして下記コマンドを実行する。

mysql
# mysql で実行できなければ以下コマンド
mysql -u root -p
# コマンド実行後、自分で設定したパスワードを入力する
# 上記2コマンドではユーザー名が異なる可能性が（設定したユーザー名 か root か）

Linuxへのインストール

  1. XAMPP 公式サイトから .run ファイルをダウンロードする

  2. ダウンロードしたディレクトリに移動して、以下のコマンドを実行する

    chmod +x xampp-linux-x64-x.x.xx-x-installer.run
sudo ./xampp-linux-x64-x.x.xx-x-installer.run

  3. 以下コマンドを実行

    # Webサーバーのルートディレクトリのパーミッションを変更する
sudo chmod 777 /opt/lampp/htdocs
# 「php.ini」のパーミッションを変更する
sudo chmod 777 /opt/lampp/etc/php.ini

  4. XAMPP のスタートと停止

    # スタート
sudo /opt/lampp/lampp start
# 停止
sudo /opt/lampp/lampp stop

    公開ディレクトリは /opt/lampp/htdocs

リモートホストとローカルホスト

  • リモートホスト
    インターネット上のサーバーのこと。
    プログラムを修正するたびにインターネット上のサーバーへプログラムを置くのは手間がかかる。
    また致命的なエラーの時に、同じサーバーにプログラムを置いている他人に迷惑がかかる。
  • ローカルホスト
    PHPでプログラムを書く時は、自分のパソコンにApache HTTP ServerなどのサーバーソフトウェアをインストールしてWebサーバー機能を持たせる。
    そのパソコンでPHPのプログラムを書き、ブラウザから自分自身のWebサーバーにアクセスする。
    インターネットを経由せず、自分のパソコンの中だけでクライアントとサーバーのやりとりが完結している状態である。
    こうすることで開発スピードが上がるとともに、致命的なエラーが出ても自分のパソコン内の出来事なので誰にも迷惑をかけない。
    このように自分のパソコン内に配置したWebサーバーのことをローカルホストと呼ぶ。
    自分のパソコンにWebサーバー機能を持たせることを「ローカルホストを立てる」という。

サーバーの仕事と動作

リクエストは、そのWebページを持つサーバー(通信や処理を行うコンピュータ, 受け取ったリクエストを解釈し返答であるレスポンスを返す,Server:給仕人)に到着する。
サーバーではさまざまな処理を経てHTMLをレスポンスとしてする。

   リクエスト    -----------------------------------------------------------------
---------------> |   Webサーバー       ---------------->    プログラムファイル   |
<--------------- |                                                  |            |
   レスポンス    |       ^                                          |            |
                 |       |                                          |            |
                 |       | 実行結果 (HTML)                          |            |
                 |       |                                          |            |
                 |       |                                          |            |
                 |                                                  |            |
                 | PHP実行エンジン   <-------------------------------            |
                 |   |                                                           |
                 |   |                                                           |
                 |   -----プログラムファイルに書かれた通りに処理する             |
                 |                                                               |
                 ------------------------ サーバー -------------------------------

Webサーバー : クライアントとやりとりを行う機能
PHP実行エンジン : 処理を行う機能

リクエストを受け取っているのは、Webサーバーと呼ばれる機能である。
Webサーバーはリクエストの内容から、どのファイルをレスポンスとして送り返せば良いか解釈する。

リクエストの判別

URLが「～.html」で終わっているファイル (HTMLファイル) がリクエストされた時には、数あるファイルの中から該当するHTMLファイルを取り出してレスポンスとする。
レスポンスにはHTMLや画像を送ることになっているので、見つけたHTMLファイルをそのままクライアントに送る。
このようなそのまま送れば良いHTMLのことを静的Webページという。
URLが「～.php」で終わっている場合には、PHP実行エンジンと呼ばれる機能に該当するプログラムファイルを渡す。
PHP実行エンジンは、ファイルの内容を解釈して処理を行い、HTMLを作成してレスポンスとして送信する。
このような処理を経てレスポンスとして返ってくるWebページのことを動的Webページという。
静的Webページがいつも決まったHTMLをレスポンスとするのに対し、動的WebページはPHP実行エンジンの処理結果のページである。

クライアント(ブラウザ)の仕事

ブラウザは、リクエストを送ったりレスポンスを受け取ったりするだけでなく、レスポンスとして受け取ったHTMLを解釈し、Webページとして表示を行う機能を持つ。
この世にたくさんのブラウザが存在するが、HTMLの解釈に違いがあったり、ブラウザごとに独自の機能があったりする。
Webページを見たり作成したりする場合には、文字コードが重要になってくる。
ブラウザは見た目の表示だけで無く、文字を正しく表示する仕事も受け持っている。
