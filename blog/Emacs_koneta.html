正規表現とバックスラッシュ

最重要概念！
Emacs Lisp では正規表現は「文字列」として表記する。
この引数として与えられる「文字列」は関数に利用される前に「評価」されるため、正規表現を記述する際は「評価した後の形」をイメージすればよい。

（通常の関数ではなく）正規表現エンジンを介する際の注意点
Emacs Lisp の正規表現エンジンに渡されたバックスラッシュは、メタ文字の意味を打ち消すメタ文字を意味する。
定義されてないメタ表現であれば、バックスラッシュの後の文字そのものを意味する。
例えば (re-search-forward "\\t") としても、正規表現には \t というメタ表現はないため、このバックスラッシュは無視され、't' だけに合致することになる。
\t を表現したければ タブを打ち消すための \ と、この \ を打ち消すための \ が必要となる。
すなわち、「正規表現として与える文字列」から「正規表現による解釈」までの間にバックスラッシュは「2度打ち消される」とイメージすれば良い。
（「関数に渡される前に行われる文字列の評価」と、「正規表現エンジンによって行われる評価」の二段階）

正規表現エンジンによる解釈 評価後の形（関数へ渡される形） 正規表現として与える文字列
タブ タブ \t
t \t \\t
\t \\t \\\\t
[]（文字列として） \ [ \ ] \\[\\]
.（文字列として） \. \\.
*（文字列として） \* \\*
\*（文字列として） \\\* \\\\\\*
\ 表現（正規表現における特別な意味） \文字 \\文字

文字列を評価する簡単な方法としては、*scratch* バッファに insert することが挙げられる。
eg. (insert "\\\\\\*") → \\\*
cf. (re-search-forward "\\\\\\*") → \* を検索

参考資料 : http://www.mew.org/~kazu/doc/elisp/regexp.html

The Region

region-beginning : この関数は、リージョンの先頭位置を、(整数として)リターンする。これは、ポイントかマークのいずれか小さいほうの位置である。
region-end : この関数は、リージョンの終端位置を、(整数として)リターンする。これは、ポイントかマークのいずれか大きいほうの位置である。
use-region-p : この関数は、Transient Markモードが有効でマークがアクティブであり、かつバッファー内に有効なリージョンがあればtをリターンする。

文字列内のテキスト属性とbuffer-substring-no-properties

文字列は, 文字そのものに加えて, 文字の属性も保持できる

buffer-substring-no-properties start end
この関数はbuffer-substringと同様であるが、 テキスト属性をコピーせずに文字だけをコピーする点が異なる。

unless

unless condition forms…
これはthen-formがない、ifの変種です:

(unless condition a b c)

は以下と完全に等価です

(if condition nil
  a b c)

エラーの発生の検知 (ignore-errors)

ingore-errors
エラーが返るとそのブロックからは nilが返る

(let ((result (ignore-errors
                (do-something)
                t)))
  (unless result
    (do-failed-case)))

do-somethingで errorが発生すると、エラーの発生を抑制し、 resultが nilになります。
成功するとそのままブロックの処理を続け tとなります。
あとはその結果に応じて失敗時の処理を行えば良いわけです。

変更内容による Emacs キーバインディングの設定方法

  • 既存のコマンドを引数を与えないで割り当てる -> そのまま

    (define-key global-map "\C-ca" 'fill-region)

  • 既成のコマンドに引数を与えて割り付ける -> 説明が必要 (ラムダ式で十分)

    (define-key global-map "\C-ca" 
  '(lambda ()
     (interactive)
     (backward-delete-char 2)))

  • 自作のコマンドを作成して割り付ける -> 説明が必要 (コマンドとは何かなど)

    (define-key global-map "\C-ca" 'my-command)
;;
(defun my-command ()
  (interactive)
  (insert "a"))

メジャーモードとマイナーモード

メジャーモードは一つのバッファに対して同時に一つしか設定できない。
一方マイナーモードはメジャーモードとも他のマイナーモードとも併用することができる。
文章を自動的に折り返す auto-fill-mode や overwrite mode 等が代表的な副モードである。

アドバイス定義 (defadvice)

定義済みの関数に処理を付け足すことができる。

(defadvice 定義済み関数 (class 自分で決めた仮の関数名)
  処理)
;; 処理を付け足した関数を有効にする
(ad-activate '定義済み関数)

class では、 after か before か around を指定する。

包囲アドバイス

包囲アドバイスにより, 関数の元定義を包み込むLisp式を書けます.
このとき関数の元定義を実行する場所を特別なシンボルad-do-itで指定します.
包囲アドバイスの本体に現れたこのシンボルは, 元定義（と内側の包囲アドバイス本体）のフォームを含んだprognで 置き換えられます.

(defadvice foo (around foo-around)
  "Ignore case in `foo'."
  (let ((case-fold-search t))
    ad-do-it))

これは, fooの元定義を実行するときに 大文字小文字を区別しないで探索することを保証します.

ad-do-it は実際には変数ではないが, 変数のように用いる. 関数の元定義を実行する場所を指定する.
包囲アドバイスでad-do-itを用いなければ, 関数の元定義を実行しません. これは, 元定義を完全に無効にする手段です.
