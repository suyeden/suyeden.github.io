ホームディレクトリのディレクトリ名を英語にする

~/.config/user-dirs-dirs を以下のように編集。

XDG_DESKTOP_DIR="$HOME/Desktop"
XDG_DOWNLOAD_DIR="$HOME/Downloads"
XDG_TEMPLATES_DIR="$HOME/Templates"
XDG_PUBLICSHARE_DIR="$HOME/Public"
XDG_DOCUMENTS_DIR="$HOME/Documents"
XDG_MUSIC_DIR="$HOME/Music"
XDG_PICTURES_DIR="$HOME/Pictures"
XDG_VIDEOS_DIR="$HOME/Videos"

実際の（現在の）ディレクトリ名も日本語から英語にリネームする。

mv $HOME/デスクトップ $HOME/Desktop
mv $HOME/ダウンロード $HOME/Downloads
mv $HOME/テンプレート $HOME/Templates
mv $HOME/公開 $HOME/Public
mv $HOME/ドキュメント $HOME/Documents
mv $HOME/音楽 $HOME/Music
mv $HOME/画像 $HOME/Pictures
mv $HOME/ビデオ $HOME/Videos

ロイグインし直すと設定が反映される。

デスクトップ下部のパネルの設定

高さ22ピクセル、アイコンのサイズ22ピクセルにすると、ウィンドウの下部分が隠れなくなると思う。

タッチパッド・タッチスクリーンの設定

mkdir /etc/X11/xorg.conf.d
# 40-libinput.conf は、コピーして変更してもいいし、新たに作成しても良い
# cp /usr/share/X11/xorg.conf.d/40-libinput.conf /etc/X11/xorg.conf.d/40-libinput.conf

必要であれば以下のコマンドを実行する。
（最新版では synaptics（synclient はこれに含まれる） ではなく libinput が使われるようになった。）

apt remove xserver-xorg-input-synaptics
apt install xserver-xorg-input-libinput

/etc/X11/xorg.conf.d/40-libinput.conf に、以下のように記述する。

Section "InputClass"
        Identifier "libinput touchpad catchall"
        MatchIsTouchpad "on"
        MatchDevicePath "/dev/input/event*"
        Driver "libinput"
  Option "Tapping" "on"
EndSection

Section "InputClass"
        Identifier "libinput touchscreen catchall"
        MatchIsTouchscreen "on"
        MatchDevicePath "/dev/input/event*"
        Driver "libinput"
  Option "Ignore" "on"
EndSection

uimツールバーの設定

デフォルトでは uim-toolbar-gtk が起動し、デスクトップ右下に小さいパネルが出て少し邪魔である。これをシステムトレイに格納する。

  1. uim-toolbar-gtk-systray をデフォルトで起動するように設定を変更する

    # alternatives に登録する
sudo update-alternatives --install /usr/bin/uim-toolbar uim-toolbar /usr/bin/uim-toolbar-gtk3-systray 100
# 起動のデフォルト設定を変更する
# 以下コマンド実行後、/usr/bin/uim-toolbar-gtk3-systray の番号を選択してリターンする
sudo update-alternatives --config uim-toolbar
# 登録を解除するときは以下を実行
# sudo update-alternatives --remove uim-toolbar /usr/bin/uim-toolbar-gtk3-systray

  2. uim-toolbar-gtk-systray のツールバー表示設定を行う
    デフォルトのままだと、システムトレイに格納されているトレイアイコンが潰れて表示されてしまう。
    これは、1つのアイコンの中にツールバーアイコンを全部表示しようとしてしまうため起こる。
    以下のようにして入力モードアイコンのみを表示させる。

    1. uim-pref-gtk を起動する。（uim-pref-gtk3 も必要か？）
    2. 「グループ」の中から「ツールバー」を選択する。
    3. 「入力方式切り替えメニュー」および「ボタン」ボックスの中のすべてのチェックボックスのマークを外す。
    4. 「グループ」の中から、使用する入力メソッド（Anthy、Anthy(UTF-8)、または Mozc）を選択する。
    5. 「ツールバー」ボックスの「有効にするボタン」の右端にある「編集」ボタンをクリックする。
    6. 「入力モード」のみ有効にする。
    7. 「OK」をクリックして設定ウィンドウを閉じる。

    トレイアイコンは "あ"（ひらがな入力） または "-"（直接入力） になっている。

かな入力・直接入力の切り替えキーの設定

uim-pref-gtk （「メニュー」 -> 「入力メソッド」 からも起動可）において、かな入力・直接入力の切り替えキーの設定もできるため、これを設定する。
「全体キー設定1」の「[全体]オン」および「[全体]オフ」を編集する。（「hiragana-katakana」を [全体]オン に加える。）

Chrome のインストール

  1. Google Chrome の公式ダウンロードページを開き、「64bit .deb(Debian/Ubuntu版)」を選択し、「同意してインストール」をクリック。
  2. ダイアログが開くので、「ファイルを保存する」を選択し、「OK」をクリックする。

  3. ダウンロードしたディレクトリに移動し、以下コマンドを実行。

    sudo apt install -y ./google-chrome-stable_current_amd64.deb

    しばらく待つと、インストールが完了する。

電源管理・画面スリープの設定

いくつか方法がある。

  • xfce4-Power-Manager による設定
    LXDE でも、xfce4 用の PowerManager が機能するのでこれを入れる。

    sudo apt install xfce4-power-manager

    xfce4-Power-Manager を立ち上げて（「メニュー」 -> 「電源管理」）設定を行う。

  • 設定ファイル編集による設定

    1. autostart ファイルの編集
      /etc/xdg/lxsession/LXDE/autostart を開き、末尾に以下を追記。

      @xset s off
@xset s noblank
@xset -dpms

      スクリーンセーバーやディスプレイの省電力運用の無効化を行っている模様。

    2. lightdm.conf ファイルの編集
      /etc/lightdm/lightdm.conf を開き、末尾に以下を追記。

      [SeatDefaults]
xserver-command=X -s 0 -dpms
    3. 必要に応じてスクリーンセーバーの設定もGUIで行う。
    4. 再起動すると画面スリープが無効になっているはず。

ブランク・サスペンド・ハイバネート・シャットダウン・ディスプレイの電源を切る・画面をロック・スリープ について

ディスプレイのみについて言っているのか、作業内容のメモリへの保存等ディスプレイ以外の要素も含めているのか、に注目する必要がある。
ブランクは、ディスプレイの電源を切ったり、画面を黒一色にしたり、意味はケースバイケース。画面をロックするかどうかは設定次第。
サスペンドはスリープのこと。起動中のPCの状態を保持したままハードウェアへの給電を抑える。
ハイバネートは休止状態のこと。起動中のPCの状態をストレージに保存して電源を「オフ」にし、次回PC起動時に保存した状態をストレージから復帰させる。
