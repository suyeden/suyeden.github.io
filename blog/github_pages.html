GitHub Pages とは

GitHub が持つ静的なウェブページをホスティングする機能
ウェブぺージを Git/GitHub を使って簡単に作成できる

GitHub Pages 作成の手順

  1. ユーザサイト用リポジトリの作成
    GitHub にてページ右上の「+」をクリックし、「New Repository」をクリック
  2. 「Repository name」欄に「ユーザー名.github.io」と入力し、「Create repository」ボタンをクリック
    この際「Description」は任意で入力、「Public」にチェック、「Initialize this repository with a README」にチェックする
  3. 以下3通りの方法によりローカルリポジトリを用意

    • 全く新しいリポジトリを作成する場合
      「username.github.io」ディレクトリを作成し、README.mdファイルを作成。

      # username.github.io

This is my Github Pages site.

      のように README.md に書き込み、以下の操作を行う。

      # 以下ローカルリポジトリの準備操作
git init
git add README.md
git commit -m "Initial commit"
# 以下リモート情報の追加操作
# 現在のブランチを master と名付ける
git branch -M master
git remote add origin https://github.com/username/username.github.io.git
git push -u origin master

    • すでに作成したリポジトリをプッシュする場合
      ローカルリポジトリがすでに存在している場合、以下のようにしてリモート情報を追加する

      git remote add origin https://github.com/username/username.github.io.git
# 現在のブランチを master と名付ける
git branch -M master
git push -u origin master

      必要に応じて以下のように README.md に書き込み、commit -> push する

      # username.github.io

This is my Github Pages site.

    • git clone

      git clone https://github.com/username/username.github.io
cd username.github.io

      とし、必要に応じて README.md を更新
      最初の push の時は以下のように行うことで、次からは git push だけで同じことを実施できるようになる

      # git branch -M master が必要かも
git push -u origin master

GitHub Pagesを使ったWebページでREADME.mdが初期表示されないようにする

  1. .githubフォルダ（フォルダ名のドットを忘れずに）を作る。
  2. そこにREADME.mdを突っ込む。
  3. Webページでindex.htmlが表示されるようになる。
  4. もちろん、GitHubではREADME.mdが表示される。
