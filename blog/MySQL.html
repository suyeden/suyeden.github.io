暗号化

MySQL と PHP のどちらからも利用できる暗号化関数として sha1 がある。
PHP からユーザーデータを登録する場合は passwordhash という関数でより良いパスワードを生成できる。

COUNT 関数

COUNT は MySQL が持っている関数の一つで、() 内に指定したカラムのうち、値が NULL でないものの数を数える。
() 内が * の時は格納されている値が NULL かどうかに関係なくデータの行数を取得する。

SELECT COUNT(*)
FROM テーブル名

テーブルの作成

CREATE TABLE テーブル名(
  id INT NOT NULL AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  title VARCHAR(255),
  body TEXT NOT NULL,
  date DATETIME NOT NULL,
  pass CHAR(4) NOT NULL
)DEFAULT CHARACTER SET =utf8;

「USE データベース名」したデータベースに対してテーブルを作成する。
() の中は、そのテーブルのカラムの設定で、カラム名、型、制約、属性の順に書く。
AUTOINCREMENT は整数の型に付与できる属性で、自動採番を表す。
PRIMARY KEY は主キーであることを表している。
最後に、閉じカッコの後で「DEFAULT CHARACTER SET=utf8」として、このテーブルで標準的に使う文字コードに UTF-8 を指定する。

ユーザの作成

root はすべてのデータベースを操作できるユーザであるためセキュリティ上好ましくない。
1つのデータベースだけを操作できるユーザを作成し、PHP でアクセスする際はそのユーザーでデータベースにアクセスするほうが、他のデータベースを不正利用されず安心。

GRANT 権限 ON データベース.テーブル名 to 'ユーザ名'@'ホスト' IDENTIFIED BY 'パスワード';

// ユーザを作成したら、MySQL 内部の一時データを削除するため、以下のように入力。
FLUSH PRIVILEGES;

以下例文

GRANT ALL ON tennis.* to 'tennisuser'@'localhost' IDENTIFIED BY 'password';
// GRANT ALL : すべての権限。このデータベースに対して自由に操作できる
// ON tennis.* : tennis データベースのすべてのテーブルを操作できる
// 'tennisuser'@'localhost' : 指定したユーザ名が存在しないときは、新しく作られる。既に存在する場合は権限の付与を行う。
                              @ 以降のホスト名は、データベースがどこにあるのかを示している。

ユーザーの削除

DROP USER user [, user] ...;

作成したユーザの確認

mysql -u ユーザー名 -p

にてパスワードを入力して、間違いがなければ MySQL にログインできる。

USE でデータベースを使える状態にし、「DESC テーブル名」とすることで作成したテーブルの構造が表のようになって表示される。

テーブルにレコードを新規追加

INSERT INTO テーブル名 (カラム1, カラム2, カラム3...) VALUES (値1, 値2, 値3...)

SELECT文によるデータの取得

SELECT 取得してくるカラム（全部なら *）
FROM 取得したいテーブル名
ORDER BY 並び替えの軸にするカラム DESC（ASC なら昇順・日付なら古い順、DESC なら降順・日付なら新しい順）
LIMIT 0, 10 （0件目から10件分取得する）

テーブル設計の例

カラム名 カラム型（長さ） NOT NULL 制約 説明
id INT ○ 書き込みの通し番号。主キー。自動採番にする。
name VARCHAR(255) ○ 書き込みを行った人の名前。長さは255文字まで。
title VARCHAR(255) - 書き込みのタイトル。長さは255文字まで。タイトルがないときは、（無題）と表示する。
body TEXT ○ 本文
date DATETIME ○ 書き込み日時
pass CHAR(4) ○ 4桁の数字で削除パスワード

主キーとは、そのテーブルの中でレコードを識別する一意な値のこと。
NOT NULL 制約とは、「そのカラムに必ずデータを入れる」という約束を示すもの。

MySQL のデータ型

  • 数値型

    型 型の名称 利用例
    INT 整数型 連番、個数など
    DOUBLE 倍精度浮動小数点数型 体重など、小数点を含むもの

  • 日付・時刻型

    型 型の名称 利用例
    DATE 日付型 日付
    DATETIME 日付時刻型 日付と時刻

  • 文字列型

    型 型の名称 利用例
    CHAR 固定長文字列型 郵便番号（0 から始まることがある）など、桁数が決まっているもの
    VARCHAR 可変長文字列型 名前、住所など
    TEXT 文字列型 本文など長い文章

リレーショナルデータベース（RDMS）

データを表のような形で保存し、共通項で表同士を関連付けているデータベース。つまりいくつかの表が集まってできている。
表のことをテーブル、列をカラム、行をレコードと呼ぶ。
